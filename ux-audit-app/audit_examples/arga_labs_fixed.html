Root Cause Auto-detected

A slow database query in the orders table caused cascading timeouts across the checkout pipeline. The query performed a full table scan due to a missing index on created_at.

Average query time spiked from 42ms → 988,505ms at 10:42 AM after a bulk data import removed and did not recreate composite index idx_orders_user_date.

Suggested Fix Patch Ready

Recreate the composite index on orders(user_id, created_at). This will allow the query planner to use an index seek instead of a full scan, bringing latency back to baseline.

-- Run against production replica first to confirm plan
CREATE INDEX CONCURRENTLY idx_orders_user_date
  ON orders (user_id, created_at DESC);

-- Verify: query time should drop to < 50ms
EXPLAIN ANALYZE SELECT * FROM orders
  WHERE user_id = $1 ORDER BY created_at DESC LIMIT 20;
